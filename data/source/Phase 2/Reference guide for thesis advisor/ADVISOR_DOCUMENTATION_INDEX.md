## Key Concepts Summary

### 1. Two Pathways
- **Task-Based**: For organizations without defined roles (2-way comparison)
- **Role-Based**: For organizations with defined roles (3-way comparison)

### 2. Aggregation Method
- Always use **MEDIAN** for current competency levels
- Why: Returns valid levels, robust to outliers

### 3. Decision Thresholds
- **>60% users** need more â†’ Recommend new strategy
- **20-60% users** need more â†’ Add supplements
- **<20% users** need more â†’ Note for future

### 4. Output
- ONE unified training plan per organization
- Not separate plans per role
- Includes recommendations and future pipeline

### 5. Data Requirements
- Minimum 70% employee completion rate
- Only uses latest assessment per employee
- Requires Phase 1 completion first

## Critical Design Principles

1. **Statistical Validity**: Median aggregation, minimum data requirements
2. **Organizational Evolution**: Task-based orgs evolve to role-based
3. **Practical Implementation**: One plan, clear objectives, actionable output
4. **Future Planning**: Tracks gaps for next iteration
