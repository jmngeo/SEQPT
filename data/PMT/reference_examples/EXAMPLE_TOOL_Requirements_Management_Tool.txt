1. TOOL OVERVIEW
================================================================================

The Requirements Management Tool allows teams to:
- Create and manage requirements in a structured database
- Establish traceability between requirements levels
- Track requirement status and changes
- Generate reports and traceability matrices
- Collaborate with team members

System Requirements:
- Web browser: Chrome, Firefox, or Edge (latest versions)
- Network connection to requirements server
- Valid user account with appropriate permissions

================================================================================
2. GETTING STARTED
================================================================================

--------------------------------------------------------------------------------
2.1 LOGGING IN
--------------------------------------------------------------------------------

Step 1: Open your web browser
Step 2: Navigate to: https://requirements.yourcompany.com
Step 3: Enter your username and password
Step 4: Click "Login" button

If you see "Invalid credentials" error:
- Verify CAPS LOCK is off
- Reset password via "Forgot Password" link
- Contact IT Help Desk if issues persist

--------------------------------------------------------------------------------

--------------------------------------------------------------------------------
2.3 PROJECT SETUP
--------------------------------------------------------------------------------

To create a new project:

Step 1: Click "Projects" in top menu
Step 2: Click "New Project" button
Step 3: Fill in project details:
        - Project Name: [Your project name]
        - Project Code: [Short code, e.g., PRJ001]
        - Description: [Brief description]
Step 4: Click "Create"

To configure requirement types:

Step 1: Open Project Settings (gear icon)
Step 2: Select "Requirement Types" tab
Step 3: Add types as needed:
        - User Requirement (prefix: UR-)
        - Engineering Requirement (prefix: ER-)
        - Test Case (prefix: TC-)
Step 4: Click "Save Changes"

================================================================================
3. CREATING REQUIREMENTS
================================================================================

--------------------------------------------------------------------------------
3.1 ADDING A NEW REQUIREMENT
--------------------------------------------------------------------------------

Method 1: Using the Toolbar

Step 1: Select the folder where you want to add the requirement
Step 2: Click the "+" button in the toolbar
Step 3: Select requirement type (e.g., "User Requirement")
Step 4: System assigns next available ID automatically (e.g., UR-042)
Step 5: Enter requirement text in the editor
Step 6: Click "Save" (Ctrl+S)

Method 2: Right-Click Context Menu

Step 1: Right-click on a folder in the Navigation Tree
Step 2: Select "New Requirement" from menu
Step 3: Select type and enter details
Step 4: Click "Save"

Method 3: Import from Document

Step 1: Click "File" > "Import"
Step 2: Select file format (Excel, Word, CSV)
Step 3: Browse to your file
Step 4: Map columns to requirement fields
Step 5: Click "Import"
Step 6: Review imported requirements for accuracy

--------------------------------------------------------------------------------
3.2 SETTING ATTRIBUTES
--------------------------------------------------------------------------------

Each requirement has standard attributes:

Attribute        | Description                  | Example Values
-----------------|------------------------------|------------------
ID               | Auto-generated identifier    | UR-001, ER-042
Title            | Short name                   | "Login timeout"
Description      | Full requirement text        | "The system shall..."
Status           | Workflow state               | Draft, Review, Approved
Priority         | Importance level             | High, Medium, Low
Owner            | Responsible person           | john.smith
Rationale        | Why this is needed           | "Customer request #123"

To edit attributes:

Step 1: Select the requirement
Step 2: Click "Edit" button (or double-click)
Step 3: Modify fields as needed
Step 4: Click "Save"

--------------------------------------------------------------------------------
3.3 ORGANIZING IN FOLDERS
--------------------------------------------------------------------------------

Creating a folder:
Step 1: Right-click parent folder
Step 2: Select "New Folder"
Step 3: Enter folder name
Step 4: Click "Create"

Moving requirements:
Step 1: Select requirement(s) to move
Step 2: Drag to destination folder, OR
Step 3: Right-click > "Move to..." > Select destination

Recommended folder structure:
  Project Root
  +-- User Requirements
  |   +-- Functional
  |   +-- Non-Functional
  |   +-- Compliance
  +-- Engineering Requirements
  |   +-- Hardware
  |   +-- Software
  |   +-- Interface
  +-- Test Cases
      +-- Unit Tests
      +-- Integration Tests
      +-- System Tests

================================================================================
4. ESTABLISHING TRACEABILITY
================================================================================

--------------------------------------------------------------------------------
4.1 CREATING LINKS
--------------------------------------------------------------------------------

To link requirements:

Step 1: Select the source requirement
Step 2: Click "Link" button in toolbar
Step 3: Search for target requirement by ID or keyword
Step 4: Select the target requirement
Step 5: Choose link type (see section 4.2)
Step 6: Click "Create Link"

Alternative: Drag-and-drop linking
Step 1: Hold Ctrl key
Step 2: Drag source requirement to target
Step 3: Select link type in popup
Step 4: Click "Confirm"

--------------------------------------------------------------------------------
4.2 LINK TYPES
--------------------------------------------------------------------------------

Link Type        | Meaning                           | Example
-----------------|-----------------------------------|---------------------------
Derives From     | Child derived from parent         | ER-001 derives from UR-001
Satisfies        | Requirement satisfies another     | Design satisfies requirement
Verified By      | Test verifies requirement         | TC-001 verifies ER-001
Related To       | General relationship              | UR-001 related to UR-005
Conflicts With   | Contradiction (needs resolution)  | UR-002 conflicts with UR-003

--------------------------------------------------------------------------------
4.3 VIEWING TRACE MATRIX
--------------------------------------------------------------------------------

To generate a traceability matrix:

Step 1: Click "Reports" > "Traceability Matrix"
Step 2: Select source type (e.g., User Requirements)
Step 3: Select target type (e.g., Engineering Requirements)
Step 4: Click "Generate"

Matrix View:
              | ER-001 | ER-002 | ER-003 | ER-004 |
--------------+--------+--------+--------+--------|
UR-001        |   X    |   X    |        |        |
UR-002        |        |   X    |   X    |        |
UR-003        |        |        |   X    |   X    |
UR-004        |        |        |        |   X    |

X = Link exists between requirements

================================================================================
5. COMMON TASKS
================================================================================

--------------------------------------------------------------------------------
5.1 EXPORTING REQUIREMENTS
--------------------------------------------------------------------------------

Export to Excel:
Step 1: Select folder or requirements to export
Step 2: Click "File" > "Export" > "Excel"
Step 3: Choose fields to include
Step 4: Click "Export"
Step 5: Save the .xlsx file

Export to Word:
Step 1: Select folder or requirements
Step 2: Click "File" > "Export" > "Word Document"
Step 3: Select template (if available)
Step 4: Click "Export"

--------------------------------------------------------------------------------
5.2 RUNNING REPORTS
--------------------------------------------------------------------------------

Available Reports:
- Requirements Summary: Count by type, status, priority
- Traceability Matrix: Links between requirement types
- Orphan Report: Requirements with no links
- Change History: Modifications over time
- Coverage Report: Verification status

To run a report:
Step 1: Click "Reports" in main menu
Step 2: Select report type
Step 3: Set filters (date range, folder, status)
Step 4: Click "Run Report"
Step 5: View on screen or export to PDF/Excel

--------------------------------------------------------------------------------
5.3 BASELINE CREATION
--------------------------------------------------------------------------------

A baseline captures a snapshot of requirements at a point in time.

To create a baseline:
Step 1: Click "Baselines" > "Create Baseline"
Step 2: Enter baseline name (e.g., "Design Review 1")
Step 3: Enter description
Step 4: Select scope (All or selected folders)
Step 5: Click "Create Baseline"

To compare baselines:
Step 1: Click "Baselines" > "Compare"
Step 2: Select two baselines to compare
Step 3: Click "Compare"
Step 4: View differences (added, deleted, modified)

7. TROUBLESHOOTING
================================================================================

Problem: "Connection Lost" message
Solution:
- Check your network connection
- Refresh the browser (F5)
- Log out and log back in
- Contact IT if issue persists

Problem: Cannot save changes
Solution:
- Check if another user has the item locked
- Verify you have write permissions
- Check for validation errors in red

Problem: Slow performance
Solution:
- Close unused browser tabs
- Clear browser cache
- Use filters to reduce displayed items
- Contact administrator for server status

Problem: Missing requirements after import
Solution:
- Check import log for errors
- Verify column mapping was correct
- Check if requirements were imported to different folder
